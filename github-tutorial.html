Overview

Git is a version control system which originated as a way to help teams of developers work collaboratively on code without going insane. It tracks changes to a set of files called a repository.

GitHub provides a free home for these repositories on the internet and has evolved into a neat community of not just software people, but data scientists and other nerds who like to contribute to open-source projects. Not only can you store your files for free (under a certain size), but you can also host a website (like this one) and track progress on a team project.

Terms

GitHub uses a lot of jargon terms for otherwise straightforward actions:

  • Clone or download: This is your 'download' button. You can either download the repository as a .zip or, if you have GitHub Desktop installed, set it up on your machine as a local version of the repository.
  • Commit to master/Push to origin: This is your 'save' button. When you use this on GitHub Desktop, you are first telling it to commit your local changes, and then telling it to overwrite the online copy of the repository with your changes. Leaving a detailed commit message lets other users know what changes you made and why.
  • Issues: This is basically a to-do list to which anyone can contribute.
  • Branches: Git allows projects to have multiple, independent branches where the idea is to merge the useful changes back into the 'master' version.
  • Pull requests: This is a formal proposal that says, "here are the changes I would like to make to this repository." It can be linked to specific Issues and preserves the discussion of changes, line-by-line.
  • Others: There are ton of other functions built into Git and GitHub, most of which are for specific programming-related purposes not relevant to us. I will update this list when I discover good uses for them.
